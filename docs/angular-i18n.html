Angular I18N

Since version 19.0.13, Native Federation for Angular supports Angular's built-in internationalization system, enabling multi-language Micro Frontend applications.

Setup

To use Angular I18N with Native Federation, keep the following in mind:

1. Add I18N to your shell

Make sure you add I18N support to your shell application:

ng add @angular/localize --project shell

2. Configure in angular.json

Configure I18N settings in your angular.json file. Do not use command-line parameters for I18N configuration, as the Native Federation Builder does not forward them to the ApplicationBuilder by design.

Always configure I18N in angular.json rather than via CLI flags. The Native Federation Builder will not pass CLI-based I18N parameters through to the underlying ApplicationBuilder.

3. Configure federation manifest for production

In production, ensure your federation.manifest.json points to the correct language-specific versions of your remotes. Each locale build of a remote will have its own deployment URL.

Example Configuration

Your angular.json should include the I18N configuration:

{
  "projects": {
    "shell": {
      "i18n": {
        "sourceLocale": "en",
        "locales": {
          "de": "src/locale/messages.de.xlf",
          "fr": "src/locale/messages.fr.xlf"
        }
      }
    }
  }
}

And your production federation manifest should reference locale-specific remote URLs:

{
  "mfe1": "https://cdn.example.com/mfe1/de/remoteEntry.json"
}
