Build Your Own Adapter

Status: TODO / work in progress. This page is a skeleton aimed at people who want to wire Native Federation into a bundler the project doesn't ship an adapter for. Expect sharp edges and missing detail — contributions welcome on GitHub.

Native Federation Core doesn't ship a bundler. Every adapter — esbuild, Angular, Vite — is a thin shim that implements a three-method contract (NFBuildAdapter) and hands its bundler's result back to the core. If you want to support Rspack, Rolldown, swc, or anything else, you write one of those shims.

Before you start

  • Read Build Process to understand the lifecycle (init → bundle → build).
  • Read Build Adapters for the contract itself.
  • Read Build Artifacts so you know exactly what the core produces from your adapter's output.
  • Keep the esbuild adapter source open in another tab as a reference implementation.

1. The contract

An adapter is any object that implements:

interface NFBuildAdapter {
  setup(name: string, options: NFBuildAdapterOptions): Promise<void>;

  build(
    name: string,
    opts?: { modifiedFiles?: string[]; signal?: AbortSignal }
  ): Promise<NFBuildAdapterResult[]>;

  dispose(name?: string): Promise<void>;
}

The core calls these in phases. Each phase has a unique name such as 'browser-shared', 'node-shared', 'browser-<pkg>', or 'mapping-or-exposed'. One persistent bundler context per name is the pattern you want — it's what makes watch-mode rebuilds cheap.

2. Phase cheat-sheet

  • browser-shared / node-shared — all build: 'default' externals for that platform in one go.
  • browser-<pkg> / node-<pkg> — externals declared with build: 'separate' or build: 'package', one phase per package group.
  • mapping-or-exposed — every exposes entry plus every shared tsconfig path.

The options.isMappingOrExposed flag tells you which family you're in. Shared externals and exposed modules usually need different bundler settings (format, platform, resolve.extensions, …).

3. The setup checklist

  1. Create a bundler context (esbuild context, Rspack Compiler, Vite dev server, …). Store it keyed by name.
  2. Map options.entryPoints to your bundler's entry syntax. Each entry has fileName (source), outName (desired output) and an optional key.
  3. Mark options.external as external — never inline them.
  4. Respect options.outdir, options.platform, options.dev, options.hash, options.chunks.
  5. Wire options.tsConfigPath into your bundler's TypeScript pipeline.
  6. Honor options.mappedPaths so shared tsconfig paths resolve correctly.

4. The build checklist

  1. Run the stored context. If opts.modifiedFiles is present, you're being asked for an incremental rebuild.
  2. Thread opts.signal through to anything cancellable — the core uses it to abort stale builds when a newer change lands.
  3. Return every emitted file as an NFBuildAdapterResult with an absolute fileName. Include chunks and sourcemaps — the core filters them itself.

5. The dispose checklist

  • dispose(name) — close a single phase's context.
  • dispose() with no argument — shut everything down. The core calls this from federationBuilder.close().

8. Minimal skeleton

import type { NFBuildAdapter } from '@softarc/native-federation/domain';

export function createMyAdapter(): NFBuildAdapter {
  const contexts = new Map<string, MyBundlerContext>();

  return {
    async setup(name, options) {
      const ctx = await myBundler.createContext({
        entries: options.entryPoints,
        external: options.external,
        outdir: options.outdir,
        platform: options.platform ?? 'browser',
        sourcemap: options.dev,
        tsconfig: options.tsConfigPath,
        paths: options.mappedPaths,
      });
      contexts.set(name, ctx);
    },

    async build(name, opts) {
      const ctx = contexts.get(name)!;
      const result = await ctx.rebuild({
        modifiedFiles: opts?.modifiedFiles,
        signal: opts?.signal,
      });
      return result.outputs.map(o => ({ fileName: o.path }));
    },

    async dispose(name) {
      if (name) {
        await contexts.get(name)?.dispose();
        contexts.delete(name);
      } else {
        for (const ctx of contexts.values()) await ctx.dispose();
        contexts.clear();
      }
    },
  };
}

9. Wiring it up

import { federationBuilder } from '@softarc/native-federation';
import { createMyAdapter } from './my-adapter';

await federationBuilder.init({
  options: {
    workspaceRoot: process.cwd(),
    outputPath: 'dist/mfe1',
    federationConfig: 'mfe1/federation.config.js',
    tsConfig: 'tsconfig.json',
  },
  adapter: createMyAdapter(),
});

await myAppBundler.build({ external: federationBuilder.externals });

await federationBuilder.build();
await federationBuilder.close();
