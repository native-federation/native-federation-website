Angular Localization

Native Federation supports Angular's locale data loading. Since version 20.0.6, this works out of the box when using the ignoreUnusedDeps feature.

Recommended: ignoreUnusedDeps

Since version 20.0.6, loading Angular locales works automatically when the ignoreUnusedDeps feature is activated in your federation config:

module.exports = withNativeFederation({
  // ...
  features: {
    ignoreUnusedDeps: true,
  },
});

For new projects, this setting is generated by default. No additional configuration is needed.

Fallback: shareAngularLocales

Before version 20.0.6, or when ignoreUnusedDeps is not activated, you need to explicitly share the locale files you need using the shareAngularLocales helper:

const {
  withNativeFederation,
  shareAll,
  shareAngularLocales,
} = require("@angular-architects/native-federation/config");

module.exports = withNativeFederation({
  // ...
  shared: {
    ...shareAll({
      singleton: true,
      strictVersion: true,
      requiredVersion: "auto",
    }),
    ...shareAngularLocales(["en", "de", "fr"]),
  },
});

This helper was introduced in version 19.0.14. It ensures only the specified locale files are bundled and shared, avoiding unnecessary downloads.

Automatic Shell Reloading

Native Federation provides automatic shell reloading when remote Micro Frontends finish rebuilding during development. This eliminates manual page refreshes and significantly improves the development experience when working with multiple applications simultaneously.
