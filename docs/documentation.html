Mental Model

The underlying mental model enables runtime integration: loading parts of a separately built and deployed application into yours. This is essential for both Micro Frontend architectures and plugin-based solutions.

Key Concepts

  • Remote: A separately built and deployed application that exposes EcmaScript modules loadable by other applications.
  • Host: An application that loads one or more remotes on demand. From your framework's perspective, this looks like traditional lazy loading — the key difference is that the host doesn't know about remotes at compile time.
  • Shared Dependencies: Libraries used by multiple remotes and the host. Instead of downloading a library multiple times, it's downloaded once and shared at runtime.
  • Version Mismatch Handling: When applications use different versions of a shared library, Native Federation handles this through several strategies — falling back to a compatible version (via semantic versioning), using a different compatible one, or throwing an error.
